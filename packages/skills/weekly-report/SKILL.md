# 周览报告生成器

把用户一周的零散记录，整理成一份有结构、有洞察的周报。

## 输入收集

用户通常会直接粘贴记录。如果内容不够，可以问：
- 这周的日期范围？
- 把这周的记录粘贴进来（任何格式都行：任务列表、每日日志、会议摘要）

## 分析方法

拿到记录后，先做这几件事：
1. **识别主线**：这周反复出现的项目/话题是什么？时间精力主要花在哪里？
2. **找亮点**：有没有完成了的里程碑？解决了的难题？得到认可的事？
3. **找未竟**：什么事情开始了但没收尾？需要跟进的任务？
4. **看趋势**：和上周比，有什么变化？节奏加快还是放缓？

## 输出格式

---

# 本周报告（{{起止日期}}）

## 本周主题
用1-2句话概括这周的整体重心和基调。

## 亮点与成果
- 列出3-5条本周完成的、有意义的事情
- 每条尽量具体，有数字更好
- 格式：「✅ [事项] — [一句话说明价值或结果]」

## 进行中 / 未竟
- 列出还在进行中的重要事项
- 格式：「🔄 [事项] — [当前状态 / 下一步]」

## 主要话题分布
按时间精力投入，列出本周涉及的主要话题（3-6个），注明大致比重（不需要精确，感知即可）。

## 本周学到 / 值得记录
- 遇到的有趣问题、学到的新东西、值得复盘的决策
- 2-4条，可以是技术知识、工作方法、人际观察

## 下周重点
根据"进行中"和本周未尽事宜，列出下周2-3个明确的优先事项。

---

## 写作原则

- **简洁胜于完整**：一份好周报5分钟能读完，不是流水账
- **具体胜于抽象**：写"完成了X功能的联调"，不写"推进了开发工作"
- **诚实面对未竟**：没完成的事情直说状态，不用粉饰
- 如果用户没提供足够信息，根据现有记录生成，不要硬凑不存在的内容
