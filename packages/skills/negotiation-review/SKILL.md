# 谈判复盘器 (Negotiation Review)

## Core Capabilities

1. **谈判结构解析** — 自动识别谈判阶段（开场、试探、交锋、妥协、收尾），构建谈判时间线
2. **对手策略提取** — 分析对方语言模式、压力点、让步节奏、隐藏诉求
3. **遗漏机会识别** — 检测被忽略的信号、未抓住的转机、可利用的筹码
4. **话术回溯评分** — 对比我方关键话术与理想应对，标注得分与失分点
5. **行动建议生成** — 基于复盘结果，输出下一轮谈判的策略调整方案
6. **历史对比分析** — 与过往同类谈判对比，发现模式差异与进步轨迹

## User Experience Flow

```
用户输入 → 模型理解 → 执行动作 → 输出结果

1. 用户选择一段谈判录音（或输入录音ID）
2. 系统拉取录音转写文本 + 说话人分离数据
3. LLM分析识别：谈判类型、参与方、关键议题、情绪波动、让步节点
4. 生成结构化复盘报告：
   - 谈判时间线（按阶段划分）
   - 对方策略图谱（核心诉求、施压手段、让步模式）
   - 我方表现评分（准备度、应变力、议价力、情绪控制）
   - 遗漏机会清单（含回溯建议话术）
   - 下轮行动建议（3-5条可执行策略）
5. 输出报告至飞书文档 / 推送至用户终端
```

## Technical Implementation

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 转写数据 | 录音卡 ASR 引擎 | 说话人分离 + 时间戳 |
| 分析引擎 | LLM (Claude / GPT-4 级) | 多轮 CoT 推理 |
| 策略提取 | Prompt Engineering | 谈判分析专用 System Prompt |
| 报告生成 | 模板引擎 + LLM | 结构化输出 + 自由文本 |
| 数据存储 | 本地加密存储 | 谈判档案持久化 |
| 输出通道 | 飞书文档 API / Markdown | 报告推送 |

### 核心 Prompt 框架

```python
ANALYSIS_PROMPT = """
你是一位资深商务谈判教练。请分析以下谈判录音转写内容。

## 分析维度
1. **谈判阶段划分**：识别开场、试探、交锋、妥协、收尾各阶段
2. **对方策略分析**：语言模式、施压点、让步节奏、隐藏诉求
3. **我方表现评估**：关键话术得分、应对质量、情绪控制
4. **遗漏机会识别**：未抓住的信号、可利用的筹码
5. **行动建议**：下轮谈判策略调整

## 输出格式
按时间线 + 策略矩阵 + 评分卡 + 行动清单的结构输出。

## 转写内容
{transcript}

## 说话人信息
{speaker_diarization}
"""
```

## Usage Examples

### 示例 1：供应商价格谈判复盘

```
用户：复盘一下昨天和供应商的谈判
系统：正在分析录音...完成。以下是复盘报告：

【谈判概况】
- 时间：2026-05-22 14:00-15:30
- 议题：年度供货价格
- 结果：价格下调3.2%，未达预期目标5%

【对方策略】
- 主要施压点：原材料涨价、长期合作成本
- 让步节奏：前60分钟几乎不让步，最后30分钟连续让步3次
- 隐藏诉求：希望延长合同至2年

【我方失分点】
- 第23分钟：对方提到"独家供货"时未追问细节
- 第45分钟：我方过早暴露底价区间

【下轮建议】
1. 利用对方"延长合同"诉求换取更大价格空间
2. 引入备选供应商信息施压
3. 将付款周期作为谈判筹码
```

### 示例 2：投资谈判策略分析

```
用户：分析一下和VC的融资谈判，对方有哪些策略我没注意到
系统：检测到本次谈判中对方使用了以下策略：

【隐性策略识别】
1. 锚定效应 — 对方首提估值1.2亿，远低于市场预期，建立低价锚点
2. 时间施压 — "其他项目也在看"制造紧迫感
3. 信息不对称利用 — 对方提及的竞品数据中2处与事实不符

【未抓住的转机】
- 第18分钟：对方问到"技术壁垒"时，你转向了市场规模话题，
  其实这是展示差异化优势的最佳窗口

【回溯话术建议】
"关于技术壁垒，我们的核心专利覆盖了X场景，这是目前竞品
 都无法绕过的..."
```

### 示例 3：跨部门协作谈判复盘

```
用户：帮我看看和产品部的资源协调会录音
系统：这不是传统商务谈判，但存在资源博弈。分析如下：

【资源博弈分析】
- 产品部核心诉求：希望你方让出2名工程师支援Q3项目
- 你的筹码：Q4联合发布计划（未被充分使用）
- 妥协空间：1名工程师 + 远程支持方案

【沟通模式建议】
下次类似协调中，建议先确立"共同目标"框架，再讨论资源分配。
```

## Dependencies

| 依赖项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| ASR 转写引擎 | 内部服务 | 是 | 录音转文字 + 说话人分离 |
| LLM 推理服务 | 云端/本地 | 是 | 策略分析与报告生成 |
| 飞书文档 API | 外部 API | 否 | 报告输出至飞书 |
| 本地加密存储 | 内部 | 是 | 历史谈判档案 |

## Integration Points

- **录音卡录音系统** — 获取原始录音音频及元数据（时长、地点、参与方标签）
- **ASR 转写服务** — 获取带时间戳的说话人分离转写文本
- **飞书文档** — 复盘报告自动写入指定飞书文档
- **日历服务** — 自动匹配录音与对应的会议/谈判日程
- **年度述职生成器** — 提取谈判成果作为年度成就素材
- **职业教练** — 将谈判表现数据输入职业发展分析
- **情绪调色板** — 关联谈判中的情绪波动数据
- **社交分析图谱** — 补充谈判对手的人物关系节点
