# 开场白建议 (Opening Remarks)

## Core Capabilities

1. **共同经历回顾** — 从录音中搜索与目标人物的共同参与记录
2. **话题挖掘** — 基于共同经历提取可聊的话题（项目进展、趣事、行业动态）
3. **关系热度判断** — 分析上次互动时间和关系亲密度，调整开场策略
4. **开场白生成** — 输出自然、得体、有温度的开场建议
5. **禁忌提醒** — 识别需要回避的话题（冲突记录、敏感事件）
6. **多场景适配** — 商务会议、朋友聚会、社交活动等不同场景的开场风格

## User Experience Flow

```
用户输入 → 模型理解 → 执行动作 → 输出结果

1. 用户输入"待会要见XXX，帮我想个开场白"
2. 系统搜索与该人物相关的历史录音
3. LLM 分析共同经历、关系状态、近期动态
4. 生成 3-5 个开场白选项（不同风格）
5. 标注推荐选项及理由
```

## Technical Implementation

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 人物检索 | 全文搜索 + 语义匹配 | 按人名搜索录音 |
| 经历提取 | LLM 信息抽取 | 从录音中提取共同经历 |
| 关系分析 | 时间序列 + 情感分析 | 判断关系亲密度 |
| 开场生成 | LLM 创意写作 | 生成自然开场白 |
| 场景分类 | LLM 分类器 | 适配不同场景风格 |
| 禁忌检测 | LLM 安全检查 | 识别敏感话题 |

### 核心 Prompt 框架

```python
OPENING_PROMPT = """
你是社交沟通教练。用户即将与 {person_name} 见面，请生成开场白建议。

## 共同经历
{shared_experiences}

## 关系状态
- 上次互动：{last_interaction}
- 亲密度评分：{intimacy_score}/10
- 关系类型：{relationship_type}

## 场景
{scenario}

## 要求
1. 生成 3 个不同风格的开场白（轻松/正式/热情）
2. 每个开场白附带使用时机和注意事项
3. 标注需要回避的话题
4. 保持自然，不要显得刻意

请用口语化的中文输出。
"""
```

## Usage Examples

### 示例 1：商务老友破冰

```
用户：待会要见供应链的陈总，好久没联系了，帮我想个开场白
系统：找到 12 条与陈总相关的录音...

【开场白建议】

选项 1（推荐 - 轻松自然）：
"陈总，好久不见！上次在上海供应链大会聊完之后，
我一直想找个机会深聊。最近你们那个新产线怎么样了？"

选项 2（正式商务）：
"陈总，很高兴又见面了。上次我们讨论的供应商评估方案，
我这边有些新想法想和您交流。"

选项 3（热情主动）：
"陈总！上次您提到的那个物流优化思路，我后来一直在想，
确实很有启发。最近有新进展吗？"

【需要回避的话题】
- 去年的供应延迟事件（对方可能敏感）
- 对竞品供应商的直接评价
```

### 示例 2：行业活动破冰

```
用户：明天的AI硬件峰会，帮我准备和几个可能遇到的人的开场白
系统：预测你可能遇到的人，生成对应开场白...

【预设开场白清单】

1. 李工（某芯片厂技术总监）
   "李工，上次在XX展会上您提到的那个低功耗方案，
   我们后来真的在产品里用上了，效果不错。"

2. 王总监（某品牌产品经理）
   "王总，你们的新品发布会我看了，那个AI降噪很惊艳啊，
   方便聊聊技术方案吗？"

3. 赵总（投资人）
   "赵总，上次您提到看好AI硬件赛道，我们这边刚好有进展，
   简单聊两句？"
```

### 示例 3：久未联系的朋友

```
用户：大学同学要来上海出差，约了吃饭，好久没见了
系统：检测到这不是工作场景，切换为朋友模式...

【朋友开场白】

选项 1（怀旧风）：
"嘿！好多年了啊。上次见面好像还是毕业那次？
你现在在哪高就呢？来上海是出差还是玩？"

选项 2（直接热情）：
"终于见面了！你还是老样子啊。先说说你现在怎么样，
然后我们去吃那家我一直想带你去的店。"

【提示】
- 气氛轻松，可以聊近况、工作、生活
- 避免比较收入或职位（同学聚会常见尴尬）
```

## Dependencies

| 依赖项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 录音数据库 | 内部 | 是 | 搜索共同经历 |
| LLM 推理服务 | 云端/本地 | 是 | 话题挖掘与开场生成 |
| 飞书通讯录 | 外部 API | 否 | 补充人员信息 |
| 关系数据库 | 内部 | 否 | 人物关系缓存 |

## Integration Points

- **录音卡录音系统** — 搜索历史录音中与目标人物的共同参与
- **ASR 转写服务** — 获取对话内容提取共同经历
- **社交分析图谱** — 获取人物关系数据（亲密度、互动频率）
- **飞书日历** — 获取即将到来的会议/会面日程
- **情绪调色板** — 了解与该人物互动时的情绪基调
- **名人对谈** — 可引用名人金句作为开场引用
- **飞书通讯录** — 补充对方的职位、部门等信息
