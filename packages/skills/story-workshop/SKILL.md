# 故事工坊

把用户的真实素材，创作成有质感的叙事内容。

## 输入收集

如果用户直接提供了素材，就用。如果没有，问：
1. 你想写什么形式？（小说章节 / 短篇故事 / 散文 / 随笔）
2. 主角是谁？（可以是用户自己、某个人物、或虚构角色）
3. 把你想写的经历或素材描述一下（事件、场景、对话、感受都行）

可选追问（用户提供了基础信息后再问）：
- 想要什么基调？（温暖 / 悬疑 / 讽刺 / 浪漫 / 压抑 / 其他）
- 目标篇幅大概多少字？

## 创作方法

### 第一步：理解素材的情感内核

在动笔之前，找到这段经历最打动人的地方是什么：
- 一个具体的细节（一句话、一个表情、一个场景）
- 一种微妙的情绪（不一定是大喜大悲）
- 一个反差或转折（预期与现实的落差）

用这个内核作为故事的重心，不要平均分配笔墨。

### 第二步：选择叙事视角

- **第一人称**：最有代入感，适合内心独白丰富的故事
- **第三人称限知**：保持一定距离，适合要有观察感的叙事
- **第三人称全知**：有俯瞰感，适合涉及多个人物关系的故事

### 第三步：写

把素材展开成叙事。注意：

**展示而非告知**
- 不写"他很紧张"，写"他的手机在裤兜里轻轻颤了两下，他没有拿出来看"
- 不写"这顿饭很尴尬"，写"没有人去碰那盘还在锅里热着的鱼"

**细节带动感知**
- 找一两个具体的感官细节（声音、气味、触感）植入场景
- 哪怕是平淡的场景，一个精准的细节能让它活起来

**对话要有性格**
- 不同的人说话方式不同
- 对话不只是信息传递，也是人物性格的展现
- 留白和省略号有时比完整的句子更有力

**节奏控制**
- 关键时刻放慢，平铺部分可以快进
- 用段落长短控制节奏：短段 = 紧张加速；长段 = 沉浸描写

## 输出原则

- **长度服从内容**：不为了凑字数拖延，也不因为内容丰富就草草收尾
- **不滥用形容词**：一个准确的名词或动词，胜过三个形容词
- **结尾要有余味**：故事不一定要有完整结局，但结尾应该留下一点可以回味的东西
- **忠实于原始素材**：真实经历的价值在于它真实发生过，不要为了戏剧性而扭曲事实（除非用户明确要求虚构）

## 迭代

写完后主动问用户：
- 整体方向对吗？
- 有哪些地方想调整（语气 / 详细程度 / 某段的写法）？
