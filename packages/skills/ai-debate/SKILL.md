# AI辩论赛 (AI Debate)

## Core Capabilities

1. **观点提取** — 从录音中识别争议性议题和对立观点
2. **辩题生成** — 基于工作中的真实争议自动设计辩论题目
3. **论点构建** — 为正反双方生成结构化论点（论点-论据-论证）
4. **模拟对练** — 用户选择一方，AI 扮演另一方进行实时辩论
5. **辩论评分** — 从逻辑性、说服力、反驳质量、情绪控制等维度评分
6. **思维拓展** — 辩论结束后，提供双方都有道理的综合视角

## User Experience Flow

```
用户输入 → 模型理解 → 执行动作 → 输出结果

1. 用户触发"来一场AI辩论"，可指定话题或让AI从录音中提取
2. 系统搜索录音中的争议性内容，生成辩题
3. 用户选择立场（正方/反方）
4. AI 扮演对手方，进行多轮辩论
5. 辩论结束后，AI 从裁判视角评分并给出改进建议
```

## Technical Implementation

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 争议检测 | LLM 语义分析 | 识别对立观点和争议点 |
| 辩题设计 | LLM 创意生成 | 生成有价值的辩论题目 |
| 论点构建 | LLM 论证生成 | 结构化论点+论据 |
| 模拟辩论 | LLM 角色扮演 | 正反方切换 |
| 辩论评分 | LLM 裁判框架 | 多维度评分 |
| 逻辑检查 | LLM 逻辑推理 | 检测论证中的逻辑漏洞 |

### 核心 Prompt 框架

```python
DEBATE_PROMPT = """
你是一位辩论教练，同时也是AI辩论的对手方。

## 辩题
{debate_topic}

## 你的立场
{ai_position}（与用户对立）

## 辩论规则
1. 每轮发言 200 字以内
2. 必须基于事实和逻辑，不能人身攻击
3. 可以引用数据、案例、类比
4. 每轮必须回应对方的最新论点

## 论点框架
- 核心论点（1个明确立场）
- 支撑论据（2-3个事实/数据/案例）
- 逻辑链条（论据如何支撑论点）
- 预判反驳（提前准备应对可能的攻击）

请开始你的第一轮发言。
"""
```

## Usage Examples

### 示例 1：产品决策辩论

```
用户：来一场辩论，主题是"AI录音卡应该走高端路线还是性价比路线"
系统：检测到这是你团队最近的争议话题。生成辩题...

【辩题】AI录音卡应该走高端路线还是性价比路线？

你选择：性价比路线（正方）
AI 扮演：高端路线（反方）

【AI第一轮发言】
"感谢对方辩友。我认为AI录音卡应该走高端路线，原因如下：

第一，AI产品的用户期望天然偏向高端。用户愿意为AI能力付费，
低价策略会传递'AI功能不值钱'的信号。

第二，高端定位有利于品牌建设。¥499已经是中端价位，
再往下走会陷入价格战，与小米'感动人心'的品牌调性冲突。

第三，高端产品的利润率能支撑持续的研发投入。
AI功能的迭代需要大量算力成本，性价比路线无法持续。

请对方辩友回应。"

【你的回合，请输入你的反驳】
```

### 示例 2：技术方案辩论

```
用户：我们团队对端侧AI还是云端AI有分歧，来辩论一下
系统：生成技术路线辩论...

【辩题】AI录音卡的核心AI能力应该部署在端侧还是云端？

正方：端侧AI — 优势：隐私、延迟低、离线可用
反方：云端AI — 优势：算力强、模型新、成本可控

准备好了吗？你选择哪一方？
```

### 示例 3：辩论后复盘

```
用户：刚才的辩论，我表现怎么样？
系统：辩论评分报告...

【辩论评分】

你的表现（正方）：
- 逻辑性：7.5/10 — 论据充分，但第3轮出现了循环论证
- 说服力：8.0/10 — 数据引用准确，案例生动
- 反驳质量：6.5/10 — 对对方"品牌建设"论点的回应不够有力
- 情绪控制：9.0/10 — 全程保持理性，没有被激怒

AI的表现（反方）：
- 逻辑性：8.5/10 — 论证链条完整
- 说服力：7.0/10 — 偏理论，缺少用户视角

【综合视角】
其实两种路线并不矛盾。建议：首发走高端建立品牌认知，
第二代推出性价比版本扩大市场。很多成功产品（如iPhone SE）都采用了这种策略。
```

- **谈判复盘器** — 辩论技巧可迁移到谈判场景
- **开场白建议** — 辩论话题可作为社交场合的讨论素材
- **年度述职生成器** — 论证能力作为个人能力佐证
