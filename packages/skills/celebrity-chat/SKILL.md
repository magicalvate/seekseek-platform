# 名人对谈 (Celebrity Chat)

## Core Capabilities

1. **人格模拟引擎** — 基于公开资料模拟乔布斯、马斯克、雷军等名人思维方式
2. **会议内容理解** — 分析录音转写内容，提取核心议题和关键观点
3. **观点生成** — 以名人视角对议题发表看法、建议和批评
4. **多角色对比** — 同一议题，同时模拟多位名人的不同视角
5. **个性化互动** — 支持"追问"模式，持续以名人身份对话
6. **名人人格库** — 可扩展的人格库，支持添加自定义名人

## User Experience Flow

```
用户输入 → 模型理解 → 执行动作 → 输出结果

1. 用户完成录音后，触发"让乔布斯来点评"
2. 系统提取最新录音的核心内容摘要
3. LLM 切换至名人人格模式，基于该名人的思维特点生成回应
4. 输出名人视角的点评（文字 + 可选语音合成）
5. 用户可继续追问，维持名人角色对话
```

## Technical Implementation

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 人格建模 | LLM System Prompt | 名人思维模式、语言风格、价值观 |
| 内容理解 | LLM 摘要提取 | 从录音中提取议题 |
| 观点生成 | LLM 角色扮演 | 以名人视角生成回应 |
| 语音合成 | TTS (可选) | 模拟名人语调输出 |
| 人格库 | JSON 配置 | 名人元数据管理 |
| 对话管理 | 多轮上下文 | 维持角色一致性 |

### 核心 Prompt 框架

```python
CELEBRITY_PROMPT = """
你现在是 {celebrity_name}。

## 人格特征
- 思维方式：{thinking_style}
- 语言风格：{speech_style}
- 核心价值观：{core_values}
- 代表语录：{famous_quotes}

## 会议内容摘要
{meeting_summary}

## 任务
以 {celebrity_name} 的身份和视角，对以上会议内容发表看法。
包括：
1. 你对这个议题的第一反应
2. 如果你在场，你会提出什么挑战性问题
3. 你会给出什么建议
4. 用一句你的经典风格总结

请保持角色一致性，用 {celebrity_name} 的口吻和思维方式回应。
"""
```

## Usage Examples

### 示例 1：产品方案点评

```
用户：刚开完AI录音卡的产品评审会，让乔布斯来点评一下
系统：提取会议核心内容...切换至乔布斯模式...

【乔布斯点评】

"我听完了你们的讨论，有几个想法。

第一，你们说了很多'功能'，但没说'体验'。用户不关心你有3个还是4个麦克风，
他们只关心'这个东西能不能让我忘记录音这件事'。

第二，¥499的价格...（停顿）你们是在卖硬件还是在卖体验？如果是体验，
价格不应该是关注点。iPod当年¥399，但人们记得的是'把1000首歌装进口袋'。

第三，那个vibe coding的概念很有意思，但你们的表达太工程师了。
把它翻译成人类语言：'你的想法，说出来就是代码'。

最后一个问题：如果这个产品只能做一件事，你选什么？先做好那一件事。"
```

### 示例 2：多角色对比

```
用户：让马斯克和雷军分别点评我们的商业模式
系统：生成双视角对比...

【马斯克视角】
"订阅制是对的，但为什么是¥199/年？如果我是你，我会思考：
这个产品的边际成本能不能趋近于零？如果能，为什么要设限？
应该让用户免费用基础功能，用高级AI能力收费。这样生态才能起来。"

【雷军视角】
"硬件¥499我觉得价格厚道，但订阅¥199要考虑用户心理。
建议先送一年订阅，让用户形成习惯。小米的逻辑是，
先把用户圈进来，后面靠生态和服务赚钱。"
```

### 示例 3：追问式对话

```
用户：乔布斯，你觉得我们的AI功能应该怎么简化？
系统：（继续乔布斯角色）

"简化不是砍功能，是想清楚本质。你的AI能做什么？
转写？总结？还是创作？选一个做到极致。

想象一个场景：用户开完会，手机弹出一条消息——
'会议总结已生成，3个待办已加入日历。'
就这一句话，其他都是噪音。"
```
