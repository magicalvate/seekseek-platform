# 正念引导 (Mindfulness Guide)

## Core Capabilities

1. **压力源识别** — 从录音中分析高压场景、紧张对话、情绪波动点
2. **压力模式建模** — 追踪压力出现的时间规律（周几、什么场景、什么人）
3. **个性化引导** — 根据用户的压力特征生成定制化冥想内容
4. **呼吸练习** — 生成不同目的的呼吸练习（减压、专注、助眠）
5. **情绪追踪** — 长期追踪情绪变化，生成情绪健康报告
6. **微休息建议** — 在高压工作间隙提供 2-5 分钟的快速放松方案

## User Experience Flow

```
用户输入 → 模型理解 → 执行动作 → 输出结果

1. 用户触发"帮我放松"或"来一段正念引导"
2. 系统分析近期录音中的情绪和压力数据
3. LLM 识别当前压力状态和压力源
4. 生成个性化的正念引导内容（文字 / 语音）
5. 引导用户完成一次冥想或呼吸练习
6. 记录练习前后的情绪变化
```

## Technical Implementation

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 情绪分析 | LLM 情感分析 | 从对话中识别情绪状态 |
| 压力检测 | 语音特征 + 文本分析 | 语速、音量、用词变化 |
| 内容生成 | LLM 创意写作 | 生成冥想引导文本 |
| 语音合成 | TTS + 舒缓语调 | 输出冥想语音 |
| 时间分析 | 时间序列 | 压力规律追踪 |
| 呼吸计时 | 前端计时器 | 节奏化呼吸引导 |

### 核心 Prompt 框架

```python
MINDFULNESS_PROMPT = """
你是一位正念冥想引导师。请根据用户的压力状态生成引导内容。

## 用户压力状态
- 当前压力水平：{stress_level}/10
- 主要压力源：{stress_sources}
- 压力持续时间：{duration}
- 用户偏好：{preferences}（语音/文字、时长、风格）

## 引导内容要求
1. 开场：30秒温和引入，引导闭眼放松
2. 身体扫描：2-3分钟，从头到脚逐步放松
3. 呼吸引导：根据压力程度选择呼吸模式
   - 高压：4-7-8 呼吸法（吸4秒-屏7秒-呼8秒）
   - 中压：方框呼吸（吸4秒-屏4秒-呼4秒-屏4秒）
   - 轻压：自然呼吸 + 注意力引导
4. 意象引导：1-2分钟，使用用户熟悉的平静场景
5. 收尾：30秒，温和带回意识

## 语言风格
温和、平静、节奏慢、用词简单、避免催促
"""
```

## Usage Examples

### 示例 1：高压会议后放松

```
用户：刚开完一个很紧张的会议，帮我放松一下
系统：检测到本次会议中情绪波动明显，压力指数 8/10。

【3分钟快速放松引导】

"现在，请找一个舒适的位置坐好，轻轻闭上眼睛。

让我们先做三次深呼吸... 吸气... 屏住... 慢慢呼出...

你刚才经历了一场紧张的会议，那些压力和情绪都是正常的。
现在，让我们把它们放下。

想象你正站在一个安静的湖边，湖面平静如镜。
微风轻轻吹过，带走了你肩上的重量...

现在开始 4-7-8 呼吸... 吸气 4 秒... 屏住 7 秒... 呼出 8 秒...

... (重复 3 轮)

很好，现在慢慢睁开眼睛，感受一下当下的平静。"
```

### 示例 2：定期压力管理

```
用户：我最近压力很大，帮我做一个压力分析
系统：分析最近 2 周的录音情绪数据...

【压力分析报告】

压力水平趋势：
- 周一：6/10（常规压力）
- 周三：8/10（产品评审日）
- 周五：7/10（周报 + 复盘）
- 周末：3/10（放松状态）

主要压力源：
1. 产品评审中的技术质疑（占比 40%）
2. 跨部门资源协调（占比 30%）
3. 时间紧迫感（占比 20%）

【本周正念计划】
- 周一早上：5分钟专注呼吸（准备新一周）
- 周三下午：3分钟 4-7-8 呼吸（评审前准备）
- 周五晚上：10分钟身体扫描（周末过渡）
```
